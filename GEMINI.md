# BYOC (Bring Your Own Cloud) - Project Overview

BYOC is a workspace-scoped, multi-tenant file management platform that allows users to connect their own S3-compatible storage buckets. The platform acts as a control plane, managing metadata, permissions, and business logic, while files are stored directly in the user's cloud storage.

## Architecture

The project is structured as a monorepo:

- **`client/`**: A modern frontend built with Next.js (App Router), React 19, and Tailwind CSS 4.
- **`server/`**: A robust backend using Express 5, TypeScript, and Prisma (PostgreSQL).

### Core Technologies

- **Frontend**: Next.js, Radix UI, TanStack Query, Zustand, Motion (Framer Motion).
- **Backend**: Express, Prisma, Redis, better-auth, AWS SDK (S3), Zod, Pino.
- **Database**: PostgreSQL (managed via Prisma).
- **Storage**: User-provided S3-compatible storage (AWS S3, Cloudflare R2, MinIO, etc.).

## Getting Started

### Prerequisites

- Node.js (v20+ recommended)
- pnpm
- PostgreSQL
- Redis

### Setup

1. **Install Dependencies**:
   ```bash
   # From root
   cd client && pnpm install
   cd ../server && pnpm install
   ```

2. **Environment Variables**:
   - Copy `.env.example` to `.env` in the `server/` directory and fill in the required values.
   - Configure your PostgreSQL and Redis connection strings.

3. **Database Migration**:
   ```bash
   cd server
   npx prisma migrate dev
   npx prisma generate
   ```

4. **Running the Application**:
   - **Backend**: `cd server && pnpm dev` (Runs on port 3000 by default).
   - **Frontend**: `cd client && pnpm dev` (Runs on port 3001 by default).

## Development Conventions

### Backend Layering

The backend follows a strict modular monolith architecture with the following layers:

1.  **Router**: Maps HTTP methods and paths to controller actions and applies middleware.
2.  **Controller**: Handles request parsing, validation (Zod), and response formatting.
3.  **Service**: Contains business logic, orchestration, and calls repositories or other services.
4.  **Repository**: Encapsulates all database interactions using Prisma or raw SQL.

Location: `server/src/modules/`

### Frontend State & Data Fetching

- **Data Fetching**: Use TanStack Query (React Query) for all server-state interactions.
- **Global State**: Use Zustand for lightweight client-side state management (e.g., UI states, upload progress).
- **API Client**: Axios-based client located in `client/lib/axios.ts` and `client/lib/api.ts`.

## Key Features & Workflows

### 1. Storage Provider Onboarding
Users connect their cloud storage by providing credentials (Access Key, Secret Key, Bucket, Region). The backend verifies the connection and encrypts credentials at rest.

### 2. Browser-to-Cloud Uploads
Files are uploaded directly from the browser to the user's S3 bucket using presigned URLs generated by the backend. This ensures the backend never proxies large file bytes.

### 3. Workspace Management
Users can create multiple workspaces with different roles (Owner, Admin, Member, Viewer) and fine-grained permissions.

### 4. Share Link System
Create public or password-protected links for files. The backend manages access and generates short-lived presigned URLs for downloading or previewing shared content.

### 5. Background Jobs
Scheduled tasks (via `node-cron`) handle upload session cleanup, share link expiry, and storage provider health checks.
